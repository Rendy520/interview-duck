# MySQL InnoDB 引擎中的聚簇索引和非聚簇索引有什么区别？ - 面试鸭 | 2026最新面试题+详细答案解析

## 回答重点

在 InnoDB 引擎中，**聚簇索引**的叶子节点直接存储完整的数据行，一张表只能有一个聚簇索引，默认就是主键索引。**非聚簇索引**的叶子节点只存储索引列的值和主键，要拿到完整数据得先查主键再回表，一张表可以有多个非聚簇索引。

核心区别就是叶子节点存的东西不一样：聚簇索引存完整数据，非聚簇索引存主键值。这导致聚簇索引查询能一步到位拿到数据，非聚簇索引可能要多一次回表操作。

## 扩展知识

### B+ 树的结构细节

InnoDB 的聚簇索引和非聚簇索引底层都是 B+ 树。

![image.png](https://pic.code-nav.cn/mianshiya/question_picture/1772087337535152129/L37VLzf2_image_mianshiya.webp)

**主键索引的 B+ 树结构**：

1）非叶子节点存储主键和页号，用来快速定位到叶子节点

2）叶子节点存储完整的数据行

3）叶子节点之间有双向链表链接，方便范围查询顺序扫描

4）叶子节点内部有页目录，记录之间是单链表链接。查找时先通过页目录二分定位，再遍历链表找到具体记录

5）B+ 树只能帮助快速定位到页，而不是具体记录。页大小默认 16KB

**非聚簇索引的 B+ 树结构**：

和主键索引的差别就在叶子节点。非聚簇索引的叶子节点只存索引列的值和主键值，不存完整的数据行。

比如一张表有 id、姓名、性别、地址字段，给姓名建一个索引，这个索引的叶子节点长这样：

### 为什么主键要有序

页大小默认 16KB，数据按主键顺序排列。如果主键无序，新插入的记录可能要塞到页中间，容量满了就会触发页分裂，性能很差。所以主键最好是自增的，这样新数据永远追加到末尾，不会产生页分裂。

用 UUID 做主键就是个典型的反面教材，每次插入都可能触发页分裂，写入性能能差 3-5 倍。

### 回表的代价

通过非聚簇索引查询时，如果需要的数据不在索引里，就得拿着主键再去聚簇索引查一遍，这个过程叫**回表**。

回表的代价不小，因为聚簇索引和非聚簇索引是两棵独立的 B+ 树，回表意味着要额外走一遍 B+ 树查找。如果查询返回大量数据，每条都要回表，可能产生大量随机 I/O，性能急剧下降。

这也是为什么不推荐 `select *`。比如通过姓名查询，只需要 id 和姓名，姓名索引的叶子节点已经存了这些数据，直接返回就行。但如果用 `select *`，得把性别、地址这些不需要的字段也查出来，就得回表去主键索引取数据，白白浪费 I/O。

### 和 MyISAM 的对比

MyISAM 没有聚簇索引，它的主键索引和普通索引结构一样，叶子节点都只存数据行的物理地址。数据文件和索引文件是分开存储的，查询时先在索引里找到地址，再去数据文件里取数据。

|   特性   | InnoDB 聚簇索引 |   MyISAM 索引   |
|--------|-------------|---------------|
| 叶子节点内容 |    完整数据行    |   数据行的物理地址    |
|  数据存储  | 索引即数据，数据即索引 |    索引和数据分离    |
|  主键查询  |  一次 B+ 树查找  | B+ 树查找 + 磁盘寻址 |
|  范围查询  |  高效，数据物理连续  | 可能产生大量随机 I/O  |

InnoDB 选择聚簇索引的好处是：主键查询效率高，范围查询时数据物理上连续，能利用磁盘预读。坏处是二级索引查询多一次回表，插入时维护数据有序有开销。

## 面试官追问

#### 提问：如果一张表没有主键，InnoDB 怎么处理聚簇索引？

回答：InnoDB 会找第一个非空的唯一索引作为聚簇索引。如果连唯一索引都没有，InnoDB 会自动生成一个 6 字节的隐藏列 row\_id 作为聚簇索引。但这个隐藏列对业务不可见，查询时用不上，所以建表时最好显式指定主键。

#### 提问：为什么 InnoDB 的非聚簇索引存主键而不是存数据行的物理地址？

回答：因为聚簇索引的数据物理位置不是固定的。页分裂、页合并、数据更新都可能导致数据行的物理地址变化。如果非聚簇索引存物理地址，每次数据移动都得更新所有相关的非聚簇索引，维护成本太高。存主键的话，主键值是稳定的，数据怎么移动都不影响二级索引。

#### 提问：什么时候非聚簇索引查询不需要回表？

> 详细解答可以看：[MySQL 的覆盖索引是什么？](https://www.mianshiya.com/question/1780933295454842881)

回答：当查询的列都在索引里时，不需要回表，这叫覆盖索引。比如索引是 `(name, age)`，查询 `select name, age from t where name = 'xx'`，索引叶子节点存了 name、age、主键，完全够用，直接返回就行。EXPLAIN 的 Extra 列会显示 `Using index`，说明走了覆盖索引。
